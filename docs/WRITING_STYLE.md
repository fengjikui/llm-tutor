# 写作风格约定

这套教程不是论文复述，也不是 API 文档。每一章都要让读者完成三件事：

1. 知道这个概念为什么出现。
2. 能读懂最小实现。
3. 能跑一个实验，并解释结果为什么大致合理。

## 讲解方式

- 先讲问题，再讲结构，最后讲公式。
- 每个公式都要配一句自然语言解释。
- 代码注释要解释 shape、训练阶段和容易误解的变量。
- 避免堆名词。第一次出现的术语必须给直觉解释。
- 不为了历史完整性讲太多过时模型，除非它能帮助理解 Transformer 或 LLM。

## 实验要求

- 每个实验都必须能用一条命令运行。
- 默认参数要适合本地 CPU 快速 smoke test。
- 训练输出至少包含 loss 和一个可解释指标。
- 教程中要解释“这个实验能证明什么”和“它不能证明什么”。

## 代码风格

- 优先清晰，不优先极致性能。
- 训练循环显式写出：forward -> loss -> backward -> step。
- 后期可以使用 Hugging Face / TRL，但核心数据格式和 loss 要先手写清楚。
